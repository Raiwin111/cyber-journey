วันที่ 2 - การแก้ไขปัญหาเบื้องต้นบน Windows และ ลองเล่น Linux ไปเรื่อย

เปลี่ยนค่า DNS และแก้ไขปัญหาเครือข่าย: ได้ทดสอบการเปลี่ยนค่า DNS เพื่อจำลองสถานการณ์เน็ตเวิร์กใช้งานไม่ได้ และหาวิธีแก้ไขให้กลับมาใช้งานได้ปกติโดยใช้คำสั่ง ipconfig /flushdns และการตั้งค่า IP ด้วยตนเอง

สำรวจ Windows Services: ได้เรียนรู้และทดสอบการจัดการ "พนักงานหลังร้าน" (Services) ของระบบ เช่น การสั่งหยุด (Stop) และเริ่มใหม่ (Start) สำหรับ Windows Update และ Print Spooler เพื่อทำความเข้าใจการทำงานเบื้องหลังของ Windows

วิเคราะห์ประสิทธิภาพระบบผ่าน Task Manager: ได้ใช้ Task Manager ตรวจสอบการทำงานของโปรเซส (Processes) และดูพฤติกรรมของระบบขณะที่สั่งหยุดหรือเริ่มการทำงานของ Service ต่างๆ

เล่นคําสั่งลองคําสั่งKali linux ไปเรื่อย ลอง nmap

วันที่ 3 - ทําโปรเจ็ค ผู้ตรวจสอบระบบ (The System Auditor)
ตอนแรก จะเช็คการทํางาน process โดยดูว่าอะไรกําลังทํางานอะไรอยู่ทั้งหมด โดยที่ เอาตัวที่กินRam เยอะที่สุดขึ้นมาไว้ข้างบน
ps aux --sort=-%mem | head -n 10 โดย ส่งผลลัพธ์ทั้งหมดไปให้คำสั่ง head เพื่อตัดมาให้ดูแค่ "10 รายการแรก" เท่านั้น
พอรู้แล้วว่ามีอะไรบ้างก็จะ มาตรวจสอบว่ามีพอร์ตไหนเปิดอยู่บ้าง โดยใช้ sudo ss -tuplnp แล้วจําลองเจาะดูพอร์ตจากภายนอก โดยใช้ nmap -sV -p- 127.0.0.1 ซึ่งจะเจอ 80 เพราะStart apache2 ไว้อยู่
