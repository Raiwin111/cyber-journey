วันที่ 2 - การแก้ไขปัญหาเบื้องต้นบน Windows และ ลองเล่น Linux ไปเรื่อย

เปลี่ยนค่า DNS และแก้ไขปัญหาเครือข่าย: ได้ทดสอบการเปลี่ยนค่า DNS เพื่อจำลองสถานการณ์เน็ตเวิร์กใช้งานไม่ได้ และหาวิธีแก้ไขให้กลับมาใช้งานได้ปกติโดยใช้คำสั่ง ipconfig /flushdns และการตั้งค่า IP ด้วยตนเอง

สำรวจ Windows Services: ได้เรียนรู้และทดสอบการจัดการ "พนักงานหลังร้าน" (Services) ของระบบ เช่น การสั่งหยุด (Stop) และเริ่มใหม่ (Start) สำหรับ Windows Update และ Print Spooler เพื่อทำความเข้าใจการทำงานเบื้องหลังของ Windows

วิเคราะห์ประสิทธิภาพระบบผ่าน Task Manager: ได้ใช้ Task Manager ตรวจสอบการทำงานของโปรเซส (Processes) และดูพฤติกรรมของระบบขณะที่สั่งหยุดหรือเริ่มการทำงานของ Service ต่างๆ

เล่นคําสั่งลองคําสั่งKali linux ไปเรื่อย ลอง nmap

วันที่ 3 - ทําโปรเจ็ค ผู้ตรวจสอบระบบ (The System Auditor) #ไม่ได้บันทึก
ตอนแรก จะเช็คการทํางาน process โดยดูว่าอะไรกําลังทํางานอะไรอยู่ทั้งหมด โดยที่ เอาตัวที่กินRam เยอะที่สุดขึ้นมาไว้ข้างบน

ps aux --sort=-%mem | head -n 10 โดย ส่งผลลัพธ์ทั้งหมดไปให้คำสั่ง head เพื่อตัดมาให้ดูแค่ "10 รายการแรก" เท่านั้น

พอรู้แล้วว่ามีอะไรบ้างก็จะ มาตรวจสอบว่ามีพอร์ตไหนเปิดอยู่บ้าง โดยใช้ sudo ss -tuplnp แล้วจําลองเจาะดูพอร์ตจากภายนอก โดยใช้ nmap -sV -p- 127.0.0.1 ซึ่งจะเจอ 80 เพราะStart apache2 ไว้อยู่

วันที่ 4 - ทําโปรเจ็ค Scan Network #โปรเจ็คแรก
วันนี้ลองหาโปรเจ็คในเน็ตดูและใช้ AI ช่วย ในการลองทําถือว่าได้เรียนรู้ เราตั้งค่าให้ Network ของทั้ง kali และ window มัน เป็นแบบ Bridged เพื่อให้มันได้รับ ip เดียวกันที่อยู่วงแลนเดียวกันกับเครื่องเป้าหมาย และเราก็ปิด firewall ใน window ด้วย แล้วผลก็ออกมาตามภาพที่อยู่ในโปรเจ็คตอนแรกสแกนไม่เจอเพราะ เปิด firewallเอาจริงๆมันมี Detail รายละเอียดเยอะกว่านี้มากแต่ฉันจําไม่ได้ทั้งหมดขอโทษด้วยตัวฉัน แต่เดี๋ยวในอนาคตต้องเข้าใจมากกว่านี้

โดยที่เราจะเขียบคําสั่ง โดย ลูบให้มันอ่าน บรรทัดและ เราตั้งเงื่อนไขโดยที่ หากว่า บรรทัดนั้นน้อยกว่า 6 ให้ ขึ้นว่ารหัสนั้น ง่ายเพราะสั้นเกินไป แต่ถ้าหากว่าไม่ให้ขึ้นว่า รหัสนั้นปลอดภัยซึ่งเป็นการกรองแบบวิเคราะรหัสภายในไฟล์ที่เก็บไว้ใน password.txt

วัน่ที่ 7 - วันนี้ ทําโปรเจ็ค Scanner Pro (Network Port Scanner Script)
การพัฒนาสคริปต์สแกนเครือข่าย: ได้เขียนสคริปต์ Bash เพื่อทำหน้าที่สแกนพอร์ตเป้าหมายโดยใช้เครื่องมือ Nmap โดยมีการนำ Logic การตรวจสอบการทำงานของสคริปต์มาปรับใช้ ทั้งการเขียนผลลัพธ์ลงไฟล์ชั่วคราว การกรองเฉพาะพอร์ตที่เปิดใช้งานด้วย grep รวมถึงการจัดการการแสดงผลให้เป็นระเบียบด้วย ANSI Color Codes

การแก้ไขข้อผิดพลาดของระบบและโฟลเดอร์แชร์: ดำเนินการแก้ปัญหาการเชื่อมต่อ Shared Folder ระหว่างเครื่อง Windows และ VirtualBox โดยการติดตั้ง VirtualBox Guest Additions ซึ่งรวมถึงการแก้ไขปัญหา Kernel Headers ของระบบปฏิบัติการที่ไม่สอดคล้องกัน และการจัดการข้อผิดพลาดของแพ็กเกจด้วยคำสั่ง dpkg เพื่อซ่อมแซมระบบให้กลับมาอยู่ในสถานะสมบูรณ์

การปรับปรุงและตรวจสอบ Logic ของโปรแกรม: มีการปรับปรุงโครงสร้างของสคริปต์ให้ทำงานได้ถูกต้องตามลำดับขั้นตอน (Logical Flow) โดยเฉพาะการย้ายการรับค่า Input ของ IP Address มาไว้ที่ส่วนต้นของสคริปต์ เพื่อป้องกันการสแกนในสภาวะที่ตัวแปรยังเป็นค่าว่าง พร้อมทั้งตรวจสอบและแก้ไขข้อผิดพลาดจากการพิมพ์ตัวแปรผิด (Syntax/Typo Correction) เพื่อให้สคริปต์สามารถประมวลผลได้อย่างราบรื่น
